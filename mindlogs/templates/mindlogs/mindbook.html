A manual for exploring, logging, and cloning your neural dev history.

🧠 What is a Mind Log?

A Mind Log is a snapshot of what you’re working on, thinking about, or debugging — written in a terminal-style interface. It helps you (and others) track cognitive flow, effort, and breakthroughs in real time.

📘 Terminology

  • sig — A unique log signature (e.g., sig-e49a7c) used to identify any mind log.
  • latency — A symbolic value you assign (in ms) to reflect your mental effort or delay. More = deeper burn.
  • neurocolor — The “vibe” or tone of your log, picked via a color wheel (red = fire, purple = focus, green = flow).
  • clone-log — A way to resonate with or fork someone’s original log. Shows inspiration or shared flow.
  • username.mind.logs — A command used to trace someone’s logs via the execution bar.

✍️ How to Create a Log

  1. Go to the terminal input interface at /mind or homepage.
  2. Write what you're working on, e.g., ~$ Refactoring auth.py again 🤯
  3. Select a neurocolor that reflects your mood or mental energy.
  4. Set your symbolic latency (e.g., 270ms for intense sessions).
  5. Press Enter or hit commit to log it.

🧬 How to Clone a Log

When you feel connected to someone’s log — maybe you’re doing something similar, inspired, or just vibing — hit the ~$ clone-log button. It logs the original under your name with attribution.

Note: Even reclones track back to the first author. No confusion, just resonance.

🖥 Using the Execution Bar

Curious about what someone’s brain is working on? Use the execution bar to type: username.mind.logs — and access their public neural feed.

Try it with: adithya.mind.logs
